웹페이지 배포 방법 안내 미노

이 페이지는 다양한 방법으로 웹페이지를 배포하는 방법을 안내합니다.
각 방법의 특징과 접근 방법, 주의사항을 참고하세요.
(대학생 교육용)
🖥️
1. VSCode Live Server로 배포하기
  • VSCode에서 Live Server 확장 프로그램 설치 후, HTML 파일에서 오른쪽 클릭 → "Open with Live Server" 선택
  • 실행 PC의 내부 IP와 포트(기본 5500)로 같은 공유기 내 다른 기기에서 접속 가능
    예시: http://192.168.0.100:5500
  • 공유기 포트포워딩을 설정하면 외부에서도 공인 IP로 접속 가능
    예시: http://공인IP:5500
  • 일부 인터넷 업체는 DDNS(도메인)을 제공하며, 설정 시 도메인 주소로도 접속 가능
    예시: http://myhome1234.iptime.org:5500
내부 IP는 cmd → ipconfig로 확인, 포트포워딩은 공유기 관리자 페이지에서 설정
외부 접속 시 방화벽/보안 설정에 주의
🌐
2. Github Pages로 배포하기
  • Github에 새 저장소를 만들고, HTML/CSS/JS 파일을 업로드
  • 저장소의 Settings → Pages에서 main branch를 선택해 배포
  • 잠시 후 https://사용자명.github.io/저장소명 주소로 접속 가능
    예시: https://사용자명.github.io/deploy_html
  • 정적 웹사이트만 지원(서버 기능 필요시 사용 불가)
무료, 별도 서버 필요 없음, 전 세계 어디서나 접근 가능
변경사항 반영 시 push 후 1~2분 소요
☁️
3. AWS, 구름IDE 등 클라우드로 배포하기
  • AWS EC2 등 클라우드 서버에 HTML 파일을 업로드하고 웹서버(Apache/Nginx 등)로 서비스
  • 구름IDE는 웹 기반 개발환경에서 바로 웹페이지 실행 및 외부 공개 가능
  • 각 서비스에서 제공하는 퍼블릭 IP 또는 도메인으로 접속 가능
  • 서버/클라우드 환경이므로 동적 웹앱, 백엔드도 구현 가능
AWS는 요금 발생 주의, 구름IDE는 무료 플랜 용량 제한 있음
방화벽, 인바운드 규칙 등 네트워크 설정 필수
🍓
4. 라즈베리파이로 배포하기
  • 라즈베리파이는 리눅스 환경에서 동작하며, 실제 배포 방식은 1번(VSCode Live Server)과 동일합니다.
  • Python(Flask), Node.js, 또는 Live Server 등으로 HTML 파일을 서비스할 수 있습니다.
  • 같은 공유기 내에서는 라즈베리파이의 내부 IP:포트로 접속
    예시: http://192.168.0.50:5000
  • 공유기 포트포워딩 시 외부에서도 공인 IP:포트로 접속 가능
  • DDNS 설정 시 도메인으로도 접속 가능
내부 IP 확인: ifconfig 또는 hostname -I 명령어 사용
외부 접속 시 보안/방화벽 설정에 주의
API 테스트 (내부/외부 FastAPI 서버)
API 주소에 내부(로컬) 또는 외부(공인IP, DDNS, 클라우드) 주소를 입력해 호출할 수 있습니다.
🦙
5. Streamlit으로 배포 및 활용하기
  • Streamlit은 파이썬 코드만으로 손쉽게 웹 대시보드, 데이터 시각화, 인터랙티브 앱을 만들 수 있는 오픈소스 프레임워크입니다.
  • 실행: streamlit run 101_streamlit.py
  • 파이썬 코드 내에서 st.button, st.text_input, st.dataframe 등 다양한 위젯을 활용해 빠르게 웹앱을 제작할 수 있습니다.
  • 주로 데이터 분석 결과 공유, 머신러닝 데모, 실시간 대시보드, 간단한 내부 도구 등에 적합합니다.
  • API 방식과의 차이: Streamlit은 화면이 계속 갱신되는 구조로, 버튼 클릭/입력 등 사용자 상호작용이 있을 때마다 전체 화면이 새로 그려집니다.
  • 별도의 HTML/CSS/JS 작성 없이 파이썬만으로 웹 인터페이스를 만들고, 내부망/외부망 모두 배포할 수 있습니다.
데이터 기반 실습, 빠른 프로토타이핑, 대시보드, 실시간 시각화 등에 매우 유용합니다.
Streamlit은 화면이 계속 갱신되는 구조이기 때문에, 메모리 관리(st.session_state 등)를 잘 해야 하며, 자바스크립트(JS) 활용이 쉽지 않고 화면이 갱신될 때 JS로 만든 요소가 사라질 수 있습니다. 또한 Streamlit 함수(블록) 밖에서는 JS나 상태 변경이 불가능한 한계가 있습니다.
TIP: 목적과 상황에 맞는 배포 방법을 선택하세요.
(실습/교육/공개용 등 다양한 상황에 활용 가능)
